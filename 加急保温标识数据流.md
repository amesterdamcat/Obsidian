# 加急保温标识数据流

## 概述

`urgent_flag`（加急）和 `insulation_flag`（保温）写入 `Frecipe_ext` 字段（DDO 对应 `SetRecipeExt`），最终落地到 `db_recipe.t_user2recipe_xxx.Frecipe_ext`，通过 `QueryRecipeList` 输出到 `recipe_ext_obj2`。

> [!warning] 历史问题
> 之前错误地写入了 `recipe_cache_ext`（`SetRecipeCacheExt`），导致值落不到 `Frecipe_ext`，已修正。

## 三个写入入口

### 1. WorkStationRecipeAdd（工作台加方）

```
deal.cpp WorkStationRecipeAdd
  → 从请求读取 sUrgentFlag / sInsulationFlag
  → oRcp.SetRecipeExt(common::AlterJsonExt(...))
```

- 文件：`deal_proj/web/deal/src/deal.cpp:19116`
- 修正：`SetRecipeCacheExt` → `SetRecipeExt`

### 2. RecipeProcess（处方处理/代寄下单）

```
deal.cpp RecipeProcess
  → 从 jsonRoot 读取 urgent_flag
  → 从 jsonRoot["others_list"] 扫描 insulation_flag
  → oRcp.SetRecipeExt(common::AlterJsonExt(...))
```

- 文件：`deal_proj/web/deal/src/deal.cpp:22192`
- 新增写入

### 3. RecipeCharge（白露渠道收费）

```
dealao.cpp RecipeCharge
  → 限定 DEAL_SOURCE_BAILU1
  → 从 oReqInfo.GetExt() 读取 urgent_flag / insulation_flag
  → oRcp.SetRecipeExt(common::AlterJsonExt(...))
```

- 文件：`deal_proj/ao/dealao/src/dealao.cpp:2742`
- 新增写入
