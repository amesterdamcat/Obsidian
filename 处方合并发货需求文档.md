# **处方合并发货需求文档**

**版本**: v1.1 | **日期**: 2026-05-11

## 一、 **项目背景与目标**

### **1.1 背景**

当前多张处方独立收费、独立发运，患者需支付多笔快递首重费，且接收多个包裹体验不佳。部分患者（如同一家人）希望合并处方、一次性支付运费、统一收货。

### **1.2 目标**

1. **合并寄付**：支持当日未发运处方合并寄送，优化患者取药体验。

2. **运费优化**：主处方承担首重，副处方分摊续重，降低患者总运费。

3. **统一发货**：统一生成 1 个单号，简化物流追踪与药房管理。

4. **权责清晰**：明确主副处方关系、解绑退费规则，保障财务账目合规。

5. **代煎限制：**若主处方和副处方选择的代煎出库地点不同不能合并快递支付

## **二、核心术语与约束**

### **2.1 术语定义**

|   |   |   |
|---|---|---|
|**术语**|**定义**|**合并特性**|
|**主处方**|合并组中第一个完成支付的处方|承担合并组首重（仅限寄付）|
|**副处方**|后续绑定到主处方的处方|享受续重优惠（仅限寄付），依附于主处方|
|**寄付**|寄件人支付运费|支持**运费合并** + **物理合并**|
|**到付**|收件人支付运费|持**物理合并**|
|**包邮**|免运费|支持物理合并|

### **3.2 运费计算规则（仅寄付有效）**

l **寄付**：按总重量统一计算。主处方付首重，副处方分摊续重。最后副处方的收费计算：当前副处方重量✖️每公斤续重费

l **到付/包邮**：**无运费合并计算逻辑**，保持各处方独立计费（到付由收件人到付），但发货时物理打包为一个包裹。

### **3.3 合并管理（解绑/退费/补收）**

#### **A. 退费规则**

|   |   |   |
|---|---|---|
|**场景**|**退费状态**|**系统行为 & 资金处理**|
|**未发货：主处方退费**|**阻断性影响**|1. **拦截发货**：系统自动锁定该合并组内所有副处方的发货状态。<br><br>2. **重新计费**：副处方需重新计算运费（脱离主处方后需补收独立首重差价或成为新主处方）。<br><br>3. **补收放行**：仅当副处方完成运费补收/重算后，才解锁允许发货。<br><br>4. **主处方退费**：退药品费 + 快递费（视具体情况，若整单取消）。<br><br>5. **主处方先补收再退费：**主处方退费时提醒用户先补收快递费（产生新的快递收费单关联这次合并行为）|
|**已发货：主处方退费**|**资金保护**|不影响副处方状态，正常流转。|
|**副处方退费**|**无影响**|1. **独立退费**：直接退还该副处方费用（含其分摊的续重费）。<br><br>2. **主处方保留**：主处方及其他副处方不受影响，继续合并发货。|

#### **B. 解绑规则**

l **解绑权限**：仅副处方都可以发起（已审方的处方不能解绑）

l **解绑后果**：

¡ **寄付**：不影响主处方的费用；解绑出的副处方需**补收独立首重费**后方可发货。

¡ **到付**：仅解除发货关联，各自独立发货。

## **四、药房发货与打印**

### **4.1 发货控制**

|   |   |
|---|---|
|**场景**|**系统行为**|
|**发货提醒**|无论寄付还是到付，点击发货时均弹窗：_“合并发货处方，合并处方编号 XXX/XXX，请核对合并发货。”_|
|**拦截锁死**|**副处方****“运费待补收”**（因主处方退费导致）的状态，**禁止点击发货**，提示先处理费用。|

### **4.2 单号与打印**

l 生成 1 个快递单号。主处方打印运单号、副处方不打印运单号
