# **包材收费配置需求文档**

版本:v1.0

 日期:2026-05-13

 状态:待评审

## **一、背景与目标**

寄付下单当前调用顺丰"收费标准"接口:系统将**药品重量 + 收件地址**传给顺丰,顺丰返回快递费由患者支付。

前期遗漏了包材的处理,本次补齐两件事:

1. **包材费由我方收取**,不再让顺丰计算。

2. **包材重量并入药品重量**一起传给顺丰,作为计费重量。

## **二、核心计费逻辑**

这是本需求的核心,所有页面、配置都服务于这套规则。

### **2.1 计费重量**

![](file:///C:\Users\PC\AppData\Local\Temp\ksohtml18644\wps5.png)

l 药品重量:由处方药品汇总得出(原有逻辑)。

l 包材重量:按"区域 + 类型 + 对应剂数"匹配到的包材规则,取规则上配置的"重量"。

l **传给顺丰的重量是计费重量**,顺丰按此返回快递费。

### **2.2 包材费**

![](file:///C:\Users\PC\AppData\Local\Temp\ksohtml18644\wps6.png)

l 单价、收费单位:取自匹配到的包材规则。

l **15 剂以上拆 2 个包装,单项收费**(即包材费 = 单价 × 2)。

l 包材费由我方系统独立向患者收取,**不进入顺丰计费**。

### **2.3 患者最终快递费用**

下单时三项分列展示、一次性支付:

|   |   |   |
|---|---|---|
|**费用项**|**来源**|**入账科目**|
|快递费|顺丰接口返回(基于计费重量：药品重量+包材重量)|快递代收|
|**包材费**|**匹配规则:单价 × 收费单位数量**|**包材收入(新增)**|

### **2.4 匹配优先级与回退**

下单时按**区域 + 类型 + 对应剂数**三个维度精确匹配规则:

l **类型**:代煎 / 自煎 / 代煎 + 自煎(后者可被两种类型同时命中)。

l **对应剂数**:支持预设(7剂以内 / 7剂以上 / 膏方丸剂等)或自定义剂数区间(如 7-10 剂)。

l **15 剂以上**:系统自动按"拆 2 个包装"逻辑,匹配到对应区间规则后乘以 2。

l **未命中**:下单拦截,提示"当前区域 / 类型 / 剂数未配置包材,请联系管理员",不允许患者自行支付。

### **2.5 计费快照**

下单成功时,系统将本次匹配到的规则(规则 ID + 包材编码 + 单价 + 收费单位 + 重量 + 包材费 + 计费重量)以快照形式落库。

 后续配置变更**不追溯历史订单**,所有退费、对账、纠纷均以快照为准。

## **三、后台配置**

### **3.1 包材库(系统内置)**

系统预置包材主数据,业务侧不维护,仅做"选择"。字段:

|   |   |
|---|---|
|**字段**|**说明**|
|顺丰物料编码|与顺丰对齐的物料编码,如 430300308905|
|包装名称|完整规格名,如 "纸箱\|30cm_25cm_20cm\|标准F3\|双坑\|对开口型"|
|简称|用于打单展示,如 "F3纸箱"|
|默认重量|包材库自带的默认值,可在配置规则时覆盖|

### **3.2 包材收费规则(本次开发的配置项)**

#### **3.2.1 列表页**

字段顺序与价目表一致:**区域 / 类型 / 对应剂数 / 顺丰物料编码 / 包装名称 / 简称 / 单价 / 收费单位 / 重量 / 操作**。

####  **顶部支持按区域、类型、对应剂数、关键字筛选;操作支持编辑、删除。3.2.2 新增 / 编辑配置(4 个区块)**

**① 适用范围**

|   |   |   |
|---|---|---|
|**字段**|**必填**|**说明**|
|适用区域|必填|单选,如广州 / 深圳|
|类型|必填|代煎 / 自煎 / 代煎 + 自煎|
|对应剂数|必填|预设(7剂以内 / 7剂以上 / 膏方丸剂...)或自定义区间(如 7-10 剂,含边界)|

**② 选择包材**(从内置包材库选,不允许新建)

|   |   |
|---|---|
|**字段**|**说明**|
|包材|必选,从包材库检索|
|顺丰物料编码 / 简称 / 包装名称|选中后自动带出,不可改|

**③ 收费与重量**

|   |   |   |
|---|---|---|
|**字段**|**必填**|**说明**|
|单价|必填|元 / 收费单位|
|收费单位|必填|个 / 套 / 袋|
|重量|必填|默认取包材库默认值,可按本条规则覆盖。该值会并入计费重量传顺丰|

#### **3.2.3 唯一性校验**

同一组**区域 + 类型 + 对应剂数 + 生效时间**有效区间内只能有一条规则。新增 / 编辑时若冲突,保存拦截并提示冲突规则。

## **四、退费规则**

|   |   |
|---|---|
|**场景**|**包材费**|
|未发货整单退|全退|
|未发货部分退(不影响剂数档位)|不退,按整单计|
|已发货退费|**不退**|

合并寄付场景:包材费按处方独立计、独立退,不参与首重续重分摊;详见《合并处方寄付需求文档》。

## **五、异常处理**

|             |                         |
| ----------- | ----------------------- |
| **异常**      | **处理**                  |
| 未命中规则       | 拦截下单,提示联系管理员            |
| 命中规则但包材库已停用 | 拦截下单 + 推送管理员告警          |
| 顺丰接口异常      | 沿用现有兜底逻辑;包材费独立核算,不阻塞展示  |
| 配置变更时点订单    | 以**下单时刻**匹配,以快照为准,不追溯历史 |
