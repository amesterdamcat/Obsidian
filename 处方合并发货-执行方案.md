## Step 6：副订单下单后加入快递合并组

**位置**：`WorkStationRecipeAdd` 下单成功（`TransState` 完成）后，仅 `iIsMergeShip == 1` 时执行。

**作用**：把主副订单所有处方加入同一个 `UnirecipeGroup`，审方时 `CreateExpressBillProxy` 自动让整组共享一个 `mail_no`，无需修改审方流程。

```cpp
if (ptrExArgs != NULL && ptrExArgs->iIsMergeShip == 1) {
    // 构建 recipe_ids 数组：主订单处方 + 副订单处方
    Json::Value rcpIds(Json::arrayValue);
    for (size_t i = 0; i < ptrExArgs->vMasterRecipes.size(); ++i) {
        rcpIds.append(Json::Value(
            lexical_cast<std::string>(ptrExArgs->vMasterRecipes[i].GetCloudRecipeId())
        ));
    }
    for (size_t i = 0; i < vecRcp.size(); ++i) {
        rcpIds.append(Json::Value(
            lexical_cast<std::string>(vecRcp[i].GetCloudRecipeId())
        ));
    }

    Json::Value body;
    body["recipe_ids"] = rcpIds;

    // 调 Go recipe-service API
    // POST /recipe/handler/InsertUnirecipeGroup
    CCUrlHttpClient oClient;
    oClient.Init();
    CHttpRequest oReq;
    CHttpResponse oRsp;
    oReq.SetUrl(CStringUtils::Format("%s/recipe/handler/InsertUnirecipeGroup", m_goapiurl.c_str()));
    oReq.SetTimeout(5);
    oReq.SetPostBody(JsonHelper::toStr(body));
    int iRet = oClient.Post(oReq, oRsp);
    // 失败只记日志，不阻断下单流程（mail_no 共享是审方时的事，下单阶段非阻塞）
    if (iRet != 0) {
        C2C_WW_LOG_ERR(iRet, "InsertUnirecipeGroup fail: %s", oClient.GetLastError().c_str());
    }
    tLog << "|InsertUnirecipeGroup|masterRcpCnt:" << ptrExArgs->vMasterRecipes.size()
         << "|subRcpCnt:" << vecRcp.size();
}
```

> [!note] 快递单号存储
> `mail_no` 由 `CreateExpressBillProxy` 写入处方的 `Fdeliver_no`（`db_recipe.t_user2recipe_xxx`），订单侧无对应字段，无需在 deal_ext 维护。
